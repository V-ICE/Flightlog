# UAVLogBook — Docker Setup

## Architecture

```
                    ┌─────────────────────────────────────┐
  Browser  ──────▶  │  Nginx :80/:443  (production only)  │
                    │  Rate limiting · SSL · Caching       │
                    └─────────────┬───────────────────────┘
                                  │  proxy_pass
                    ┌─────────────▼───────────────────────┐
                    │  PHP 8.2 + Apache :80               │
                    │  API (index.php) · React SPA        │
                    │  Video streaming · Log parsing      │
                    └──────┬──────────────┬───────────────┘
                           │ PDO          │ Volume mount
                    ┌──────▼──────┐  ┌───▼────────────────┐
                    │  MySQL 8.0  │  │  Named Volumes      │
                    │  (internal  │  │  uploads/logs/      │
                    │   network)  │  │  uploads/videos/    │
                    └─────────────┘  └────────────────────-┘
```

## Quick Start

### 1. Configure

```bash
cp .env.example .env
# Edit .env — set passwords, JWT secret, domain name
nano .env
```

Minimum required changes in `.env`:
- `MYSQL_ROOT_PASSWORD` — strong password
- `MYSQL_PASSWORD` — strong password
- `JWT_SECRET` — 64 random chars (`openssl rand -hex 32`)

### 2. Start (development)

```bash
make dev
# or manually:
docker compose -f docker-compose.yml -f docker-compose.dev.yml up
```

| Service | URL |
|---------|-----|
| Web app (React) | http://localhost:3000  (Vite HMR) |
| API | http://localhost:8080/api/v1 |
| MySQL | localhost:3306 (TablePlus, DBeaver etc.) |

### 3. Start (production, no SSL)

```bash
make up
# or:
docker compose up -d
```

App available at `http://localhost:8080`

### 4. Start (production, with Nginx + SSL)

```bash
# Start full stack with Nginx
make prod
# or:
docker compose --profile production up -d

# Get Let's Encrypt certificate (domain must point to this server)
make ssl D=yourdomain.com E=admin@yourdomain.com
```

App available at `https://yourdomain.com`

## What happens on first start

1. **MySQL** starts and auto-imports `schema.sql` (all tables created)
2. **Builder** (Node) runs `npm ci && npm run build` → React app compiled into `web_dist` volume
3. **App** (PHP+Apache) waits for MySQL health check to pass, then starts serving
4. Subsequent starts: builder skips build (dist already exists), full stack ready in ~5 seconds

## Common commands

```bash
make logs           # tail all container logs
make logs-app       # tail PHP/Apache logs only
make shell          # bash shell inside app container
make db-shell       # MySQL shell
make backup         # dump database to ./backups/
make restore F=backups/file.sql
make rebuild-web    # force React rebuild (after web/src changes)
make rebuild-app    # rebuild PHP Docker image (after Dockerfile changes)
make ps             # show running containers
make down           # stop everything
make clean          # DESTRUCTIVE: remove containers + volumes
```

## Data persistence

All runtime data lives in Docker **named volumes** (survives `docker compose down`):

| Volume | Contents |
|--------|----------|
| `uavlogbook_db_data` | MySQL database files |
| `uavlogbook_uploads_logs` | Uploaded flight log files |
| `uavlogbook_uploads_videos` | Uploaded FPV video files |
| `uavlogbook_web_dist` | Built React app (regenerated by builder) |
| `uavlogbook_ssl_certs` | SSL certificates (production) |

Only `make clean` or `docker volume rm` deletes them.

## Environment variables reference

| Variable | Default | Description |
|----------|---------|-------------|
| `MYSQL_ROOT_PASSWORD` | — | **Required.** MySQL root password |
| `MYSQL_PASSWORD` | — | **Required.** App DB user password |
| `JWT_SECRET` | — | **Required.** 32+ char random string |
| `ANTHROPIC_API_KEY` | _(empty)_ | Claude API key for AI import (optional) |
| `VITE_API_URL` | `/api/v1` | API URL baked into React build |
| `ALLOWED_ORIGINS` | `http://localhost,...` | Comma-separated CORS origins |
| `NGINX_SERVER_NAME` | `localhost` | Domain for Nginx SSL |
| `APP_PORT` | `8080` | Host port for direct app access |
| `PHP_MEMORY_LIMIT` | `256M` | PHP memory limit |
| `PHP_UPLOAD_MAX_FILESIZE` | `8192M` | Max upload size |

## Deployment on a VPS / cloud server

```bash
# On the server:
git clone https://your-repo/uavlogbook.git
cd uavlogbook
cp .env.example .env && nano .env   # set real secrets
make prod                            # start with Nginx
make ssl D=yourdomain.com E=you@email.com   # get SSL cert
```

Works on any server with Docker + Docker Compose installed:
- Ubuntu 22.04 / 24.04
- Debian 12
- Amazon Linux 2
- Any VPS (DigitalOcean, Hetzner, Linode, AWS EC2)
